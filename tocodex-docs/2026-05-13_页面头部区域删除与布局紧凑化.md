# 页面头部区域删除与布局紧凑化备忘录

## 变更摘要

按照要求，全局移除了业务路由内容区顶部的页面级头部区域，包括页面标题、页面说明以及右侧的刷新/返回等辅助按钮。删除后页面核心内容直接上移展示，工单回收站页面现在由 Tabs 下方直接进入搜索与表格卡片，不再保留“工单回收站 / 查询已逻辑删除的工单... / 刷新”区域。

同时，核心业务操作按钮未直接丢弃，已移动到对应搜索/筛选工具区中；非核心辅助按钮（如刷新、返回）已删除，以提升后台页面的紧凑性和表格可视空间。

## 主要处理策略

- 删除统一页面头部结构 `page-header` / `page-title` / `page-description`。
- 删除全局样式中已不再使用的页面头部相关样式，避免后续误用。
- 保留搜索、筛选、表格、分页、恢复、编辑、删除等核心业务能力。
- 将核心新增入口移动到筛选区域右侧：
  - 工单列表：`新建工单`
  - 用户管理：`新增用户`
  - 部门管理：`新增部门`
  - 角色管理：`新增角色`
  - 菜单管理：`新增菜单`
- 删除页面头部中的辅助刷新/返回按钮。
